# Whats91 Reports API LLM Guide

Use this Markdown brief as implementation context for LLMs that need to generate or debug Whats91 reporting integrations.

## Project Context

- Product: Whats91 Developer Docs.
- API family: Public API v2 only.
- Reports base URL: `https://graph.whats91.com/api/v2/reports`.
- Reports APIs are read-only JSON endpoints.
- Do not use `/v2/reports`; use `/api/v2/reports`.
- Reports are useful for message delivery status, campaign performance, recipient history, template analytics, and failure diagnosis.
- Webhooks provide real-time delivery events; Reports query stored state.
- Sender selection uses `senderId`, which is the WhatsApp sender phone number.
- If a token is number-scoped or resolves one default sender, `senderId` can be omitted.

## Authentication

Use server-side bearer token authentication.

```http
Authorization: Bearer w91_live_xxx
```

GET endpoints can also accept `authToken`, `auth_token`, or `token` query parameters for compatibility, but the recommended approach is the `Authorization` header.

Do not expose Whats91 bearer tokens in browser or mobile frontend code. Call Whats91 from a trusted backend service.

## Common Query Parameters

- `senderId`: WhatsApp sender phone number.
- `page`: Positive integer. Default `1`.
- `limit`: Positive integer. Default `50`, max `200`.
- `dateFrom`: Start date in `YYYY-MM-DD`.
- `dateTo`: End date in `YYYY-MM-DD`.
- `storage`: `live` or `archive`. Default is usually `live`.
- `status`: Message status such as `accepted`, `sent`, `delivered`, `read`, `failed`, or `pending`.
- `sourceType`: Source such as `PUBLIC_API`, `CAMPAIGN`, `CHATBOT`, or `FLOW`.

## Endpoint Index

### Message Reports

- `GET /api/v2/reports/messages`: List message reports with pagination and filters.
- `GET /api/v2/reports/messages/status`: Retrieve latest status and lifecycle events for one message.
- `GET /api/v2/reports/messages/history`: List message history for one recipient mobile number.

### Campaign Reports

- `GET /api/v2/reports/campaigns`: List campaign summaries and delivery totals.
- `GET /api/v2/reports/campaigns/{campaignUid}`: Retrieve one campaign summary with totals and breakdowns.
- `GET /api/v2/reports/campaigns/{campaignUid}/messages`: List recipient-level message rows for a campaign.
- `GET /api/v2/reports/campaigns/{campaignUid}/timeline`: Retrieve campaign timeline events.
- `GET /api/v2/reports/campaigns/responses`: Search and filter customer responses generated by campaign interactions.

### Analytics Reports

- `GET /api/v2/reports/analytics/delivery`: Delivery summary by status and source.
- `GET /api/v2/reports/analytics/failures`: Failure analytics grouped by error and template/source.
- `GET /api/v2/reports/analytics/templates`: Template-wise delivery, read, and failure counts.
- `GET /api/v2/reports/analytics/dates`: Date-wise message counts and status counts.

## Node.js Fetch Example

```js
const token = process.env.WHATS91_API_TOKEN;

async function listReports({ senderId, status = 'delivered', limit = 50 }) {
  const url = new URL('https://graph.whats91.com/api/v2/reports/messages');
  if (senderId) url.searchParams.set('senderId', senderId);
  url.searchParams.set('status', status);
  url.searchParams.set('limit', String(limit));

  const response = await fetch(url, {
    headers: {
      Authorization: `Bearer ${token}`,
    },
  });

  const payload = await response.json();

  if (!response.ok || payload.success === false) {
    throw new Error(payload.message || 'Report request failed');
  }

  return payload.data;
}
```

## PHP cURL Example

```php
<?php
$token = getenv('WHATS91_API_TOKEN');

$query = http_build_query([
    'senderId' => '919999999999',
    'status' => 'delivered',
    'limit' => 50,
]);

$ch = curl_init('https://graph.whats91.com/api/v2/reports/messages?' . $query);
curl_setopt_array($ch, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => [
        'Authorization: Bearer ' . $token,
    ],
]);

$raw = curl_exec($ch);
$status = curl_getinfo($ch, CURLINFO_HTTP_CODE);
curl_close($ch);

$response = json_decode($raw, true);

if ($status >= 400 || empty($response['success'])) {
    throw new RuntimeException($response['message'] ?? 'Report request failed');
}

print_r($response['data']);
```

## Common Errors

```json
{
  "success": false,
  "message": "Missing authorization token",
  "error_code": "MISSING_AUTH_TOKEN",
  "metadata": {
    "apiVersion": "v2",
    "requestId": "request-uuid"
  }
}
```

Common error codes:

- `MISSING_AUTH_TOKEN`: No bearer or query token was supplied.
- `INVALID_AUTH_TOKEN`: Token is invalid, expired, revoked, or user inactive.
- `SENDER_NOT_ALLOWED`: Number-scoped token attempted another sender.
- `FEATURE_NOT_AVAILABLE`: Subscription does not include the requested report feature.
- `VALIDATION_FAILED`: Invalid date range, pagination, storage, or missing required query field.
- `MESSAGE_NOT_FOUND`: Message status lookup could not find the message.
- `CAMPAIGN_NOT_FOUND`: Campaign detail lookup could not find the campaign.

## Practical Use Cases

### Delivery Dashboard

Use `GET /api/v2/reports/analytics/delivery` for aggregate delivery charts and `GET /api/v2/reports/messages` for drill-down rows.

### Customer Support History

Use `GET /api/v2/reports/messages/history?mobileNumber=...` to show all messages for one recipient in a support tool.

### Campaign Performance

Use campaign list, campaign detail, campaign messages, timeline, and responses to build a complete campaign reporting view.

### Failure Diagnosis

Use `GET /api/v2/reports/analytics/failures` and filter by `templateName` or `sourceType` to identify recurring delivery issues.

### Template Performance

Use `GET /api/v2/reports/analytics/templates` to compare sent, delivered, read, and failed counts by template.

## Debugging Checklist

- Confirm `Authorization: Bearer w91_live_xxx` is present.
- Confirm GET requests use the `/api/v2/reports` base path.
- Do not use `/v2/reports` or `https://api.whats91.com/v2/reports`.
- Confirm `senderId` is present when the token cannot resolve a sender.
- Confirm number-scoped tokens are not querying another sender.
- Confirm required query fields such as `messageId`, `mobileNumber`, or `campaignUid`.
- Confirm `page` and `limit` are positive integers.
- Keep `limit` at or below `200`.
- Confirm `dateFrom` and `dateTo` use `YYYY-MM-DD`.
- Use `storage=archive` when searching older report data if supported by the account.
- Check `metadata.requestId` when contacting support.
